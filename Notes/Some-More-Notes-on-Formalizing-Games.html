Some More Notes on Formalizing Games

Advantage Growth

One thing I've realized is that the correct relation between advantages is that to have a reduction f:G→G′f : G \to G'f:G→G′, you need to have:

p(λ)Adv[G′,Af]≥Adv[G,A]p(\lambda)\text{Adv}[G', \mathscr{A}_f] \geq \text{Adv}[G, \mathscr{A}]p(λ)Adv[G′,Af​]≥Adv[G,A]

for some polynomial λ\lambdaλ, for all adversaries A\mathscr{A}A, with Af\mathscr{A}_fAf​ being the wrapped adversary induced by the morphism of games.

This is the most general relation satisfying Adv[G′,AF]\text{Adv}[G', \mathscr{A}_F]Adv[G′,AF​] negligible ⟹Adv[G,A]\implies \text{Adv}[G, \mathscr{A}]⟹Adv[G,A] negligible, which is really the only notion we care about.

Naturally, this composes as well.

The Unwinnable Game

We can define an unwinnable game as one with trivial messages, where the advantage is a negligible function of λ\lambdaλ.

This has the nice property that the statement: A→0A \to 0A→0, is equivalent to saying the the game AAA is secure. This lets us derive nice things categorically, such as: A→0,B→0⟹A+B→0A \to 0, B \to 0 \implies A + B \to 0A→0,B→0⟹A+B→0, which follows directly from the universal property of the coproduct!

If you do some shenanigans, you can show that this object is initial. To do this, you need to fake messages, which you can do by simply sampling them at random, but this imposes additional assumptions on the structure of games.

Identical Games, Isomorphic Games

Two games are isomorphic in the natural category-theoretic definition. This means that there are reductions A→BA \to BA→B and B→AB \to AB→A, which compose in both ways to make the identity function.

Two games share the same interface if their message types are the same.

We can say that two games AAA and BBB are identical when they share the same interface, and their responses are always the same for each query.

One common proof of technique is to look at two games, and then assert they're actually the same game. This is usually asserting that AAA and BBB are identical, or A=BA = BA=B.

Induced Game

A morphism between games fff is really a set of data used to bridge the two interfaces together. We've used this to make an induced adversary fff, but you can also use it to make an induced game BfB_fBf​, from a morphism A→BA \to BA→B. We can rephrase things in terms of the induced game too. I.e. for a reduction, we require:

p(λ)Adv[Bf,A]≥Adv[A,A]p(\lambda)\text{Adv}[B_f, \mathscr{A}] \geq \text{Adv}[A, \mathscr{A}]p(λ)Adv[Bf​,A]≥Adv[A,A]

This is useful with the technique of identical games, because in many situations we can create a reduction A→BA \to BA→B by creating a morphism fff, and then showing that Bf=AB_f = ABf​=A. This lets us move state, like random oracles, into a wrapper we control, which is very useful.

Distinguishing Game

Given two games with the same interface, we can define the game: A⋄BA \diamond BA⋄B. We start by flipping a coin b←R{0,1}b \xleftarrow{R} \{0, 1\}bR​{0,1}, and then playing one of the two games based on that coin. We then have two conditional distributions, based on the value of this coin, and the advantage becomes:

∣AdvA(P[CA_N+1∣b=0])−Adv_B(P[CB_N+1∣b=1])∣|\text{Adv}_A(P[C^A\_{N+1}|b=0]) - \text{Adv}\_B(P[C^B\_{N+1} | b=1])|∣AdvA​(P[CA_N+1∣b=0])−Adv_B(P[CB_N+1∣b=1])∣

For example, the bit flipping version of semantic security is of this form.

Often, we formulate a security property in terms of two games, and show that A⋄B→0A \diamond B \to 0A⋄B→0

One thing that I feel is true, but haven't proven entirely rigorously is that:

A≅B⟺A⋄B→0A \cong B \iff A \diamond B \to 0A≅B⟺A⋄B→0

But this should absolutely be true.

Difference Lemma

A common technique to show A⋄B→0A \diamond B \to 0A⋄B→0 is to show that the execution of AAA and BBB are the same, except if some event ZZZ happens. This naturally creates the conditions to show that the difference in advantages is negligible, since the difference probability distributions are bounded by the likelihood of ZZZ hot happening.

I'm not sure of a generic way to prove this, as it depends a lot on the particulars of a situation, and is a very internal property. Perhaps there is a way to externalize it.

Hybrid Arguments

Often, we want to prove A⋄C→0A \diamond C \to 0A⋄C→0, but we do so in two steps, first by proving A⋄B→0A \diamond B \to 0A⋄B→0, and B⋄C→0B \diamond C \to 0B⋄C→0, and then gluing those two together. The typical way this is done is by using:

∣Adv_A−Adv_C∣≤∣Adv_A−Adv_B∣+∣Adv_B−Adv_C∣|\text{Adv}\_A - \text{Adv}\_C| \leq |\text{Adv}\_A - \text{Adv}\_B| + |\text{Adv}\_{B} - \text{Adv}\_{C}|∣Adv_A−Adv_C∣≤∣Adv_A−Adv_B∣+∣Adv_B−Adv_C∣

Usually, this hybrid argument and triangle lemma application are repeated. There's actually a way to abstract over this.

To do this, we need to use a little result:

A⋄C→(A⋄B)+(B⋄C)A \diamond C \to (A \diamond B) + (B \diamond C)A⋄C→(A⋄B)+(B⋄C)

Since they all share the same interface, there's clearly a morphism A⋄C⇢A⋄BA \diamond C \dashrightarrow A \diamond BA⋄C⇢A⋄B and A⋄C⇢B⋄CA \diamond C \dashrightarrow B \diamond CA⋄C⇢B⋄C, the question is whether or not the advantages work out. For this, note that for (A⋄B)+(B⋄C)(A \diamond B) + (B \diamond C)(A⋄B)+(B⋄C), the advantage is:

\max(∣AdvA−AdvB∣,∣AdvB,AdvC∣)\max(|\text{Adv}_A - \text{Adv}_B|, |\text{Adv}_B, \text{Adv}_C|)max(∣AdvA​−AdvB​∣,∣AdvB​,AdvC​∣)

Now clearly:

∣Adv_A−Adv_C∣≤∣Adv_A−Adv_B∣+∣Adv_B−Adv_C∣|\text{Adv}\_A - \text{Adv}\_C| \leq |\text{Adv}\_A - \text{Adv}\_B| + |\text{Adv}\_{B} - \text{Adv}\_{C}|∣Adv_A−Adv_C∣≤∣Adv_A−Adv_B∣+∣Adv_B−Adv_C∣

But we also have:

∣Adv_A−Adv_B∣+∣Adv_B−Adv_C∣≤2⋅\max(∣AdvA−AdvB∣,∣AdvB,AdvC∣)|\text{Adv}\_A - \text{Adv}\_B| + |\text{Adv}\_{B} - \text{Adv}\_{C}| \leq 2 \cdot \max(|\text{Adv}_A - \text{Adv}_B|, |\text{Adv}_B, \text{Adv}_C|)∣Adv_A−Adv_B∣+∣Adv_B−Adv_C∣≤2⋅max(∣AdvA​−AdvB​∣,∣AdvB​,AdvC​∣)

So we do indeed have a reduction, using the fact that 222 is polynomial in λ\lambdaλ. This is why it's important to have that definition of reduction!

This naturally generalized to multiple differences, instead of just two.
