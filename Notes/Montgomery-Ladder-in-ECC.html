Montgomery Ladder in ECC

This is a technique to efficiently calculate the x coordinate x(nP)\bold{x}(nP)x(nP) given the x coordinate of a point on a Montgomery curve x(P)\bold{x}(P)x(P). This method is also easily implemented in a constant-time way.

This works for curves:

By2=x3+Ax2+xBy^2 = x^3 + Ax^2 + xBy2=x3+Ax2+x

Satisfying B(A2−4)≠0B(A^2 - 4) \neq 0B(A2−4)=0.

In Summary

The ladder itself

func ScalarMul(n Scalar, x Field) Field {
  nP := Projective{1, 0}
  n1P := Projective{x, 1}

  swap := 0
  for b := range n.msb .. n.lsb {
    swap ^= b
    condSwap(swap, nP, n1P)
    swap = b

    nP, n1P = step0(x, nP, n1P)
  }
  condSwap(swap, nP, n1P)
  return nP.X / nP.Z
}

Implementing the individual step:

// In field elements of course
const A24 = (A - 2) / 4

func step0(x Field, nP, n1P Projective) (Projective, Projective) {
  double_X := nP.X + nP.Z
  double_Z := nP.X - nP.Z
  // Addition
  add_X := n1P.X + n1P.Z
  add_Z := nP.X - nP.Z
  add_X *= double_Z
  add_Z *= double_X
  tmp := add_X - add_Z
  add_X += add_Z
  add_Z = tmp
  add_X.square()
  add_Z.square()
  add_Z *= x
  // Now we can do doubling
  double_X.square()
  double_Z.square()
  // Store for later
  Xn_plus_Zn_squared := double_X
  double_X *= double_Z
  tmp = Xn_plus_Zn_squared - double_Z
  double_Z = A24 * tmp
  double_Z += Xn_plus_Zn_squared
  double_Z *= tmp
  return (
    Projective{double_X, double_Z},
    Projective{add_X, add_Z}
  )
}

This could be optimized to reuse nP and n1P's buffers better as well

Doubling Formula

Given a point P=(x,y)P = (x, y)P=(x,y), the formula for 2P2P2P is given by:

x2=Bλ2−A−2xy2=λ(x−x2)−y\begin{aligned} &x_2 = B \lambda^2 - A - 2x \cr &y_2 = \lambda (x - x_2) - y \end{aligned}​x2​=Bλ2−A−2xy2​=λ(x−x2​)−y​

where:

λ=3x2+2Ax+12By\lambda = \frac{3x^2 + 2Ax + 1}{2By}λ=2By3x2+2Ax+1​

(This formula is derived geometrically. λ\lambdaλ is the slope of the tangent line to the curve at the point PPP.)

Because λ2\lambda^2λ2 features By2By^2By2 in the denominator, we can replace that with x3+Ax2+xx^3 + Ax^2 + xx3+Ax2+x, and then calculate out the following identity:

x2=(x2−1)24(x3+Ax2+x)x_2 = \frac{(x^2 - 1)^2}{4(x^3 + Ax^2 + x)}x2​=4(x3+Ax2+x)(x2−1)2​

This formula calculates x(2P)\bold{x}(2P)x(2P) given only x(P)\bold{x}(P)x(P), and can be shown to be valid in other edge cases. See: 1.

Projective Form

If we use projective coordinates, then we have x=X/Zx = X / Zx=X/Z. Plugging this into the formula we had above, we get:

x2=(X2−Z2)24ZX(X+AXZ+Z2)x_2 = \frac{(X^2 - Z^2)^2}{4ZX(X + AXZ + Z^2)}x2​=4ZX(X+AXZ+Z2)(X2−Z2)2​

In other terms, this means we have:

X2=(X2−Z2)2Z2=4XZ(X+AXZ+Z2)\begin{aligned} &X_2 = (X^2 - Z^2)^2 \cr &Z_2 = 4XZ(X + AXZ + Z^2) \end{aligned}​X2​=(X2−Z2)2Z2​=4XZ(X+AXZ+Z2)​

If we calculate this directly, a point doubling takes:

2M+3S+2C2 \bold{M} + 3 \bold{S} + 2 \bold{C}2M+3S+2C

We can re-arrange things to allow more sharing:

X2=(X+Z)2(X−Z)2Z2=((X+Z)2−(X−Z)2)((X+Z)2+A−24((X+Z)2−(X−Z)2))\begin{aligned} &X_2 = (X + Z)^2(X - Z)^2 \cr &Z_2 = ((X + Z)^2 - (X - Z)^2)\left( (X + Z)^2 + \frac{A - 2}{4}((X + Z)^2 - (X - Z)^2) \right) \end{aligned}​X2​=(X+Z)2(X−Z)2Z2​=((X+Z)2−(X−Z)2)((X+Z)2+4A−2​((X+Z)2−(X−Z)2))​

If you note that:

(X+Z)2−(X−Z)2=4XZ(X + Z)^2 - (X - Z)^2 = 4XZ(X+Z)2−(X−Z)2=4XZ

The correctness of this formula becomes relatively clear.

This adjusted formula takes:

Ladder

One simple way to calculate kPkPkP is as follows. Iterate over the bits of kkk from most to least significant. If a bit b=0b = 0b=0, set:

P←2PP \leftarrow 2PP←2P

If b=1b = 1b=1, then set:

P←2P+PP \leftarrow 2P + PP←2P+P

Our addition formula works well when we have PnP_nPn​ and Pn+1P_{n + 1}Pn+1​, and want to calculate P2n+1P_{2n + 1}P2n+1​, since the difference is P1=PP_1 = PP1​=P, a point we already know.

At each iteration, as we rise up to PkP_kPk​, our final value, we want to keep this property, and keep the successive values around.

This, when b=0b = 0b=0, we do:

(Pn,Pn+1)↦(P2n,P2n+1)(P_n, P_{n + 1}) \mapsto (P_{2n}, P_{2n + 1})(Pn​,Pn+1​)↦(P2n​,P2n+1​)

The first value comes from point doubling PnP_nPn​, and the second from differential point addition, using the fact that the difference is P1P_1P1​.

When b=1b = 1b=1, we want to do:

(Pn,Pn+1)↦(P2n+1,P2n+2)(P_n, P_{n + 1}) \mapsto (P_{2n + 1}, P_{2n + 2})(Pn​,Pn+1​)↦(P2n+1​,P2n+2​)

We can calculate the first value through point addition, and the second through point doubling.

In fact, if we define these two functions as step0\text{step}_0step0​, and step1\text{step}_1step1​ respectively, we notice that step1\text{step}_1step1​ comes from swapping the input, and then the output. With:

swap(P,Q):=(Q,P)\text{swap}(P, Q) := (Q, P)swap(P,Q):=(Q,P)

We have:

step1=swap;step0;swap\text{step}_1 = \text{swap};\text{step}_0;\text{swap}step1​=swap;step0​;swap

Thus, an algorithm for doing our ladder would be:

for b in bits:
  if b == 1:
    swap(P, Q)
  (P, Q) = step0(P, Q)
  if b == 1:
    swap(P, Q)
return P

We can make this "constant-time", by replacing our branching with a conditional swap:

for b in bits:
  cswap(b, P, Q)
  (P, Q) = step0(P, Q)
  cswap(b, P, Q)
return P

If we have two conditional swaps chained together

cswap(bits[i], P, Q)
cswap(bits[i - 1], P, Q)

Which happens between iterations, notice that this is effectively:

cswap(bits[i] ^ bits[i - 1], P, Q)

because of this, we can effectively do:

cswap(bits[N - 1], P, Q)
step0(P, Q)
cswap(bits[N - 1] ^ bits[N - 1], P, Q)
step0(P, Q)
...
cswap(bits[1] ^ bits[0], P, Q)
step0(P, Q)
cswap(bits[0], P, Q)

Noting that 0 ^ bits[N - 1] = bits[N - 1], we can effectively do:

swap := 0
for b in bits:
  swap ^= b
  cswap(swap, P, Q)
  swap = b
  step0(P, Q)
cswap(swap, P, Q)
return P

This allows us to efficiently compute whether or not we need to swap.
