Consider the basic discrete logarithm game:

DLogb x←$FqPub():‾ return x⋅GWin(x^):‾ return b=0∧x^=x\boxed{ \begin{aligned} &\colorbox{#FBCFE8}{\large $\text{DLog}_b$ }\cr \cr &x \xleftarrow{\$} \mathbb{F}_q\cr \cr &\underline{\mathtt{Pub}():}\cr &\ \texttt{return } x \cdot G \cr \cr &\underline{\mathtt{Win}(\hat{x}):}\cr &\ \texttt{return } b = 0 \land \hat{x} = x\cr \end{aligned} }​DLogb​ ​x$​Fq​Pub():​ return x⋅GWin(x^):​ return b=0∧x^=x​​

Here, the generator is fixed as part of the configuration of the group. One natural question is whether or not allowing the adversary to choose the generator helps them. This variant with a chosen generator could be described as follows:

CLogb P,x←⊥SetP(P):‾ assert P≠0 P←P x←$Fq return x⋅PWin(x^):‾ assert P≠⊥ return b=0∧x^=x\boxed{ \begin{aligned} &\colorbox{#FBCFE8}{\large $\text{CLog}_b$ }\cr \cr &P, x \gets \bot\cr \cr &\underline{\mathtt{SetP}(P):}\cr &\ \texttt{assert } P \neq 0\cr &\ P \gets P\cr &\ x \xleftarrow{\$} \mathbb{F}_q\cr &\ \texttt{return } x \cdot P \cr \cr &\underline{\mathtt{Win}(\hat{x}):}\cr &\ \texttt{assert } P \neq \bot\cr &\ \texttt{return } b = 0 \land \hat{x} = x\cr \end{aligned} }​CLogb​ ​P,x←⊥SetP(P):​ assert P=0 P←P x$​Fq​ return x⋅PWin(x^):​ assert P=⊥ return b=0∧x^=x​​

[!note] Note: It is convenient to assume that the adversary cannot send a PPP equal to the identity point, which makes the rest of the proofs smoother. Sending the identity point also forces the adversary to guess randomly, so it's not a good strategy anyhow.

Now, clearly DLog≤CLog\text{DLog} \leq \text{CLog}DLog≤CLog. The question is, if DLog\text{DLog}DLog is hard, what can we say about CLog\text{CLog}CLog? Does being able to choose the generator help?

A little bit, at least with the techniques I've managed to gather together.

In particular, we can show that:

CLog≤DLog\text{CLog} \leq \sqrt{\text{DLog}}CLog≤DLog​

If anyone knows how to get a tighter bound, I'm all ears!

[!note] Note: I'm assuming that we're working in a group of prime order qqq, and thus that all points besides the identity are generators.

Reduction from Multi-CLog

Instead of reducing directly from CLog\text{CLog}CLog, we instead reduce from a variant where the adversary needs to solve multiple challenges on their provided generator:

CLogbn P,x1,…,xn←⊥SetP(P):‾ assert P≠0 P←P x1,…,xn←$Fq return x1⋅P,…Win(x^1,…,x^n):‾ assert P≠⊥ return b=0∧∀i∈[n] x^i=xi\boxed{ \begin{aligned} &\colorbox{#FBCFE8}{\large $\text{CLog}_b^n$ }\cr \cr &P, x_1, \ldots, x_n \gets \bot\cr \cr &\underline{\mathtt{SetP}(P):}\cr &\ \texttt{assert } P \neq 0\cr &\ P \gets P\cr &\ x_1, \ldots, x_n \xleftarrow{\$} \mathbb{F}_q\cr &\ \texttt{return } x_1 \cdot P, \ldots \cr \cr &\underline{\mathtt{Win}(\hat{x}_1, \ldots, \hat{x}_n):}\cr &\ \texttt{assert } P \neq \bot\cr &\ \texttt{return } b = 0 \land \forall i \in [n]\ \hat{x}_i = x_i\cr \end{aligned} }​CLogbn​ ​P,x1​,…,xn​←⊥SetP(P):​ assert P=0 P←P x1​,…,xn​$​Fq​ return x1​⋅P,…Win(x^1​,…,x^n​):​ assert P=⊥ return b=0∧∀i∈[n] x^i​=xi​​​

In this section, we show that CLog2≤DLog\text{CLog}^2 \leq \text{DLog}CLog2≤DLog, which gives us the aforementioned result that CLog≤DLog\text{CLog} \leq \sqrt{\text{DLog}}CLog≤DLog​, combining the two reductions.

Lemma: CLog2≤DLog\text{CLog}^2 \leq \text{DLog}CLog2≤DLog

The basic idea is that two queries are enough to learn the discrete logarithm of GGG with respect to PPP, and some point XXX with respect to PPP. We combine these two pieces of information to learn the discrete logarithm of XXX with respect to GGG.

In more detail, consider the following game:

Γ0 P←⊥X←super.Pub()r←$FqR←r⋅GSetP(P):‾ assert P≠0 P←P return R,XWin(r^,x^):‾ assert P≠⊥ return r^⋅P=R∧super.Win(x^⋅rr^)\boxed{ \begin{aligned} &\colorbox{#FBCFE8}{\large $\Gamma^0$ }\cr \cr &P \gets \bot\cr &X \gets \texttt{super}.\text{Pub}()\cr &r \xleftarrow{\$} \mathbb{F}_q\cr &R \gets r \cdot G\cr \cr &\underline{\mathtt{SetP}(P):}\cr &\ \texttt{assert } P \neq 0\cr &\ P \gets P\cr &\ \texttt{return } R, X \cr \cr &\underline{\mathtt{Win}(\hat{r}, \hat{x}):}\cr &\ \texttt{assert } P \neq \bot\cr &\ \texttt{return } \hat{r} \cdot P = R \land \texttt{super}.\text{Win}\left(\hat{x} \cdot \frac{r}{\hat{r}}\right)\cr \end{aligned} }​Γ0 ​P←⊥X←super.Pub()r$​Fq​R←r⋅GSetP(P):​ assert P=0 P←P return R,XWin(r^,x^):​ assert P=⊥ return r^⋅P=R∧super.Win(x^⋅r^r​)​​

I claim that Γ0∘DLogb=CLogb2\Gamma^0 \circ \text{DLog}_b = \text{CLog}^2_bΓ0∘DLogb​=CLogb2​

First, note that the adversary's perspective is the same. They see two uniformly random group elements (thanks to our obligation that PPP is a generator). So, it boils down to whether or not our check is equivalent First, note that the check we're supposed to be doing is that:

r^⋅P=r⋅Gx^⋅P=x⋅G\begin{aligned} \hat{r} \cdot P &= r \cdot G\cr \hat{x} \cdot P &= x \cdot G\cr \end{aligned}r^⋅Px^⋅P​=r⋅G=x⋅G​

Or, considering PPP to have discrete logarithm ppp, that:

r^⋅p=rx^⋅p=x\begin{aligned} \hat{r} \cdot p &= r\cr \hat{x} \cdot p &= x\cr \end{aligned}r^⋅px^⋅p​=r=x​

Note that we manually check the first equation, and so if that's false, we behave in the same way, by rejecting.

So, assume that r^p=r\hat{r} p = rr^p=r. In this case, p=rr^p = \frac{r}{\hat{r}}p=r^r​, and so applying that to the second equation gives us x^rr^=?x\hat{x} \frac{r}{\hat{r}} \stackrel{?}{=} xx^r^r​=?x, and so our discrete logarithm game will tell us that correctly.

■\blacksquare■

Reducing CLog to Multi-CLog

In this section, we prove that CLog≤CLognn\text{CLog} \leq \sqrt[n]{\text{CLog}^n}CLog≤nCLogn​.

The idea is simple, although unfortunately not a straight line reduction. (I should encapsulate this into one of "my" "state-separable proof gadgets")

The idea is that given an adversary A\mathcal{A}A for CLog\text{CLog}CLog, you create an adversary B\mathcal{B}B for CLogn\text{CLog}^nCLogn by repeatedly rewinding the adversary back to the point after sending PPP, you then feed them a different XiX_iXi​ each time, getting back x^1,…,x^n\hat{x}_1, \ldots, \hat{x}_nx^1​,…,x^n​ across the different forks. Because each XiX_iXi​ has the same distribution as in CLog\text{CLog}CLog, the probability of succeeding is bounded below by CLogn\text{CLog}^nCLogn, since we have nnn forked instances which need to succeed.

This gives us our result.
