Bit Flipping in State Separable Proofs

When working with State Separable Proofs, it's often easier to frame every game as distinguishing between the "real world" and the "ideal world". This allows keeping the framing consistent throughout the composition of different games / packages.

For example, the security of a PRF is defined in this real ideal paradigm, by distinguishing between the case of actually calling the PRF with a key, and the case of a genuine random function.

If you then use this PRF to make an encryption scheme, it's useful to also define security in terms of distinguishing between encrypting messages, and encrypting random ciphertexts, which helps the proofs go more smoothly when trying to reduce the security of the larger encryption scheme to the security of the smaller PRF.

Contrast this with the other common notion of encryption security, in which the adversary sends two messages each round, and has to figure out whether the challenger is consistently encrypting the first or the second message.

This begs the question: can you reduce the real-or-random notion of encryption to the left-or-right notion of encryption? Yes.

The Other Direction

In the other direction, the idea is more subtle. We can't directly translate an oracle encrypting either our message or a random message into one encrypting one of two messages.

The idea of the reduction in standard game notation is to choose the random bit for the message ourselves, inside of the shim. Then, if the adversary is able to accurately detect the bit, that tells us that we're in the case where we're not seeing random ciphertexts from the oracle.

There's actually a general lesson to be learned here.

Let's say you have two bit-indexed games, GbGG_{b_G}GbG​​ and HbHH_{b_H}HbH​​.

If we look at:

HbH∘GbGH_{b_H} \circ G_{b_G}HbH​​∘GbG​​

Then we can replace HbHH_{b_H}HbH​​ with a concrete game HHH, where the bit is flipped on initialization. Ditto with GGG.

From a technical point of view, HHH includes a special function to read this bit, called get-b\texttt{get-b}get-b. Then, given an adversary A\mathcal{A}A for HbHH_{b_H}HbH​​, whose interface doesn't include get-b\texttt{get-b}get-b, of course, we have an adversary B^\hat{\mathcal{B}}B^ which runs A\mathcal{A}A, and then checks if that bit is equal to get-b()\texttt{get-b()}get-b(), returning 111 if that's the case.

The question is about the advantage:

ϵ(B^∘(1(get-b)⊗A)∘H∘GbG)\epsilon(\hat{\mathcal{B}} \circ (1(\texttt{get-b}) \otimes \mathcal{A}) \circ H \circ G_{b_G})ϵ(B^∘(1(get-b)⊗A)∘H∘GbG​​)

By definition, we have:

ϵ:=∣P[B^→1∣bG=0]−P[B^→1∣bG=1]∣\epsilon := |P[\hat{\mathcal{B}} \to 1 | b_G = 0] - P[\hat{\mathcal{B}} \to 1 | b_G = 1]|ϵ:=∣P[B^→1∣bG​=0]−P[B^→1∣bG​=1]∣

Letting B^(x0,x1):=P[B^→1∣bH=x0,bG=x1]\hat{\mathcal{B}}(x_0, x_1) := P[\hat{\mathcal{B}} \to 1 | b_H = x_0, b_G = x_1]B^(x0​,x1​):=P[B^→1∣bH​=x0​,bG​=x1​], we can expand this to get:

ϵ=12∣B^(0,0)+B^(1,0)−(B(0,1)+B(1,1))∣\epsilon = \frac{1}{2}|\hat{\mathcal{B}}(0, 0) + \hat{\mathcal{B}}(1, 0) - (\mathcal{B}(0, 1) + \mathcal{B}(1, 1))|ϵ=21​∣B^(0,0)+B^(1,0)−(B(0,1)+B(1,1))∣

Letting A(x0,x1)\mathcal{A}(x_0, x_1)A(x0​,x1​) denote a similar quantity, we have:

ϵ=12∣(1−A(0,0))+A(1,0)−((1−A(0,1))+A(1,1))∣\epsilon = \frac{1}{2}|(1 - \mathcal{A}(0, 0)) + \mathcal{A}(1, 0) - ((1 - \mathcal{A}(0, 1)) + \mathcal{A}(1, 1))|ϵ=21​∣(1−A(0,0))+A(1,0)−((1−A(0,1))+A(1,1))∣

which we can reduce to:

ϵ=12∣A(1,0)−A(0,0)+A(0,1)−A(1,1)∣\epsilon = \frac{1}{2}|\mathcal{A}(1, 0) - \mathcal{A}(0, 0) + \mathcal{A}(0, 1) - \mathcal{A}(1, 1)|ϵ=21​∣A(1,0)−A(0,0)+A(0,1)−A(1,1)∣

Using the reverse triangle inequality, we can write this as:

ϵ≥12∣∣A(0,0)−A(1,0)∣−∣A(0,1)−A(1,1)∣∣\epsilon \geq \frac{1}{2}||\mathcal{A}(0, 0) - \mathcal{A}(1, 0)| - |\mathcal{A}(0, 1) - \mathcal{A}(1, 1)||ϵ≥21​∣∣A(0,0)−A(1,0)∣−∣A(0,1)−A(1,1)∣∣

Or, in terms of advantages, we have:

ϵ≥12∣ϵ(A∘Hb∘G0)−ϵ(A∘Hb∘G1)∣\epsilon \geq \frac{1}{2}|\epsilon(\mathcal{A} \circ H_b \circ G_0) - \epsilon(\mathcal{A} \circ H_b \circ G_1)|ϵ≥21​∣ϵ(A∘Hb​∘G0​)−ϵ(A∘Hb​∘G1​)∣

Which is actually a useful result. We can use an adversary A\mathcal{A}A which can break the inner part, and use that to get an adversary breaking GbG_bGb​.

A similar reasoning shows that:

ϵ(B^∘A∘HbH∘G)≥12∣ϵ(A∘H0∘Gb)−ϵ(A∘H1∘Gb)∣\epsilon(\hat{\mathcal{B}} \circ \mathcal{A} \circ H_{b_H} \circ G) \geq \frac{1}{2}|\epsilon(\mathcal{A} \circ H_0 \circ G_b) - \epsilon(\mathcal{A} \circ H_1 \circ G_b)|ϵ(B^∘A∘HbH​​∘G)≥21​∣ϵ(A∘H0​∘Gb​)−ϵ(A∘H1​∘Gb​)∣

Let's summarize the main result. Given an adversary A\mathcal{A}A playing against HbH∘GbGH_{b_H} \circ G_{b_G}HbH​​∘GbG​​, there exists an adversary B\mathcal{B}B such that:

ϵ(B∘Gb)≥12∣ϵ(A∘Hb∘G0)−ϵ(A∘Hb∘G1)∣\epsilon(\mathcal{B} \circ G_b) \geq \frac{1}{2}|\epsilon(\mathcal{A} \circ H_b \circ G_0) - \epsilon(\mathcal{A} \circ H_b \circ G_1)|ϵ(B∘Gb​)≥21​∣ϵ(A∘Hb​∘G0​)−ϵ(A∘Hb​∘G1​)∣

ϵ(B∘RRb)≥12∣ϵ(A∘Hb∘RR0)−ϵ(A∘Hb∘RR1)∣=12∣ϵ(A∘LLb)−0∣\epsilon(\mathcal{B} \circ \text{RR}_b) \geq \frac{1}{2}|\epsilon(\mathcal{A} \circ H_b \circ \text{RR}_0) - \epsilon(\mathcal{A} \circ H_b \circ \text{RR}_1)| = \frac{1}{2}|\epsilon(\mathcal{A} \circ \text{LL}_b) - 0|ϵ(B∘RRb​)≥21​∣ϵ(A∘Hb​∘RR0​)−ϵ(A∘Hb​∘RR1​)∣=21​∣ϵ(A∘LLb​)−0∣

And that implies the reduction.
