Chinese Remainder Theorem for Programmers

  • #algebra
  • #math
  • #programming

This is a quick post about the Chinese Remainder Theorem. Specifically, how to use it to solve a system of system of simple modular equations.

This came up today in an Advent of Code problem, and enough people were asking questions about the theorem to warrant a little blog post, in my opinion.

In brief, the CRT lets us solve a system of equations:

x≡a1modn1⋮x≡akmodnk\begin{aligned} x &\equiv a_1 \mod n_1 \cr \vdots \cr x &\equiv a_k \mod n_k \end{aligned}x⋮x​≡a1​modn1​≡ak​modnk​​

by efficiently finding the xxx that satisfies all of these constraints, provided that each of the nin_ini​ are coprime, i.e. share no common divisors.

The Modulo Operation

The behavior of mod should be pretty well understood, at least for positive numbers. mod x m means dividing x with m, and looking at what's left, so we have:

mod 220 2 == 0
mod 221 2 == 1
mod 67 16 == 3

etc.

Another way of defining this is to say that mod(x,m)mod(x, m)mod(x,m) is the difference between xxx, and the largest multiple of mmm less than xxx, i.e.

mod(x,m):=x−k⋅m\text{mod}(x, m) := x - k \cdot mmod(x,m):=x−k⋅m

where k⋅m≤xk \cdot m \leq xk⋅m≤x, and ∀k′⋅m≤x.k′≤k\forall k' \cdot m \leq x. \quad k' \leq k∀k′⋅m≤x.k′≤k.

The behavior of mod for negative numbers varies depending on the programming language. In haskell, for example, mod (-x) m is -(mod x m). But according to the mathematical definition we've just given, this wouldn't be the case.

For example mod(−1,3)=2\text{mod}(-1, 3) = 2mod(−1,3)=2, because the closest multiple of 333, below −1-1−1, would be −3-3−3, and there's a difference of 222 in order to reach −1-1−1 from there. Similarly, we would have mod(−5,3)=1\text{mod}(-5, 3) = 1mod(−5,3)=1, because the closest multiple of 333 here would be −6-6−6.

We can define a function to emulate this behavior, in languages where mod (-x) m = -(mod x m):

fullMod :: Integer -> Integer -> Integer
fullMod x m = mod (mod x m + m) m

If x is positive, then we have mod x m + m, and we take that modulo m. But, adding a multiple of m, doesn't change the modulus. This is because mod(a+b,m)=mod(mod(a,m)+mod(b,m),m)\text{mod}(a + b, m) = \text{mod}(\text{mod}(a, m) + \text{mod}(b, m), m)mod(a+b,m)=mod(mod(a,m)+mod(b,m),m), and mod(m,m)=0\text{mod}(m, m) = 0mod(m,m)=0.

If x is negative, then we end up with -(mod x m) + m, i.e. m - mod x m after the first step. This is the correct behavior, since if we have something like mod(−5,3)\text{mod}(-5, 3)mod(−5,3), we want to end up with 111, but instead we get −2-2−2 first. By adding this to 333, we end up with the 111 we wanted. We don't need to do another reduction at this point, so the extra mod _ m changes nothing.

Modular Arithmetic

If we look at all of the integers, modulo some m, we end up with the set of numbers {0,1,…,m−1}\{0, 1, \ldots, m - 1\}{0,1,…,m−1}. Since these are still integers, we can carry out addition and multiplication as usual, but remembering to take the modulus with m afterwards. This gives us a ring Z/mZ\mathbb{Z}/m\mathbb{Z}Z/mZ, or CmC_mCm​, which is quite a bit shorter.

For example, in C3C_3C3​, we have 1+2=01 + 2 = 01+2=0, 2⋅2=12 \cdot 2 = 12⋅2=1, and other possibilities.

[!note] Note: To be super pedantic, the elements of CmC_mCm​ are not {0,…,m−1}\{0, \ldots, m -1 \}{0,…,m−1}, but rather {[0],[1],…}\{[0], [1], \ldots\}{[0],[1],…}, where [x][x][x] denotes the set of integers with a given modulus xxx. In practice, these work out the same, and we won't need this level of detail.

For each integer x∈Zx \in \mathbb{Z}x∈Z, we have a corresponding modulus in CmC_mCm​. Many integers will have the same modulus. To express that two integers have the same modulus, we say:

x≡ymodmx \equiv y \mod mx≡ymodm

Usually, yyy will be reduced completely, and so we this means mod(x,m)=y\text{mod}(x, m) = ymod(x,m)=y. The subtle difference here is that mod\text{mod}mod outputs an integer in the range {0,…,m−1}\{0, \ldots, m - 1\}{0,…,m−1}, but in the equivalence equation, xxx and yyy don't necessarily have to be in this range.

As further examples, we have:

6≡0mod35≡−1mod3−5≡2mod3\begin{aligned} 6 &\equiv 0 &\mod 3 \cr 5 &\equiv -1 &\mod 3 \cr -5 &\equiv 2 &\mod 3 \end{aligned}65−5​≡0≡−1≡2​mod3mod3mod3​

One useful property is that if:

x≡ymodmx \equiv y \mod mx≡ymodm

Then:

x−y≡0modmx - y \equiv 0 \mod mx−y≡0modm

i.e. x−yx - yx−y is a multiple of mmm.

Bézout's Algorithm

We just need one more tool before we can tackle the CRT: Bézout's Algorithm. This algorithm states that if we have two integers a,b∈Za, b \in \mathbb{Z}a,b∈Z, and their greatest common divisor is \gcd(a,b)\gcd(a, b)gcd(a,b), then we can find two integers x,y∈Zx, y \in \mathbb{Z}x,y∈Z, such that:

x⋅a+y⋅b=\gcd(a,b)x \cdot a + y \cdot b = \gcd(a, b)x⋅a+y⋅b=gcd(a,b)

This is basically a suped up version of Euclid's algorithm, finding not only the greatest common divisor, but also the necessary integer factors to get the equation above.

Euclid's Algorithm

Remember that the gcdgcdgcd of two numbers should return the largest number dividing both of them.

Let's go over the idea behind Euclid's algorithm first.

Note that gcd(a,0)=a\text{gcd}(a, 0) = agcd(a,0)=a. Anything divides 000, so we just need a divisor of aaa. The largest one is just aaa itself.

Now, let's say we can decompose aaa:

a=qb+ra = qb + ra=qb+r

This uses euclidean division to find a quotient and a remainder, with r<br < br<b.

We can show that any divisor of aaa and bbb is a divisor of bbb and rrr, and vice versa.

First, let's show d∣b,d∣r⟹d∣ad | b, d | r \implies d | ad∣b,d∣r⟹d∣a.

Note that if d∣bd | bd∣b, then b=kbdb = k_b db=kb​d for some factor kbk_bkb​.

With this in hand, it's clear that if d∣b,d∣rd | b, d | rd∣b,d∣r, then we can write a=qb+ra = qb + ra=qb+r as

a=qkbd+krd=(qkb+kr)da = q k_b d + k_r d = (q k_b + k_r) da=qkb​d+kr​d=(qkb​+kr​)d

This means that d∣ad | ad∣a.

Now for d∣a,d∣b⟹d∣rd | a, d | b \implies d | rd∣a,d∣b⟹d∣r.

We can write a−qb=ra - qb = ra−qb=r, and get

kad−qkbd=rk_a d - q k_b d = rka​d−qkb​d=r

and thus d∣rd | rd∣r.

Since the common divisors of aaa and bbb are nothing more than the common divisors of bbb and rrr, \gcd(a,b)\gcd(a, b)gcd(a,b) must be \gcd(b,r)\gcd(b, r)gcd(b,r).

This gives us a recursive algorithm to calculate the \gcd\gcdgcd:

gcd :: Integer -> Integer -> Integer
gcd a b | a < b = gcd b a
gcd a 0 = a
gcd a b =
  let r = mod a b
  in gcd b r

The first clause is the swapping rule, the second the base case, with gcd(a,0)\text{gcd}(a, 0)gcd(a,0), and the third implements the recursive rule we just went over.

Extending the Algorithm

Bézout's algorithm extends this to find not only the \gcd\gcdgcd, but also two factors x,yx, yx,y such that xa+yb=\gcd(a,b)xa + yb = \gcd(a, b)xa+yb=gcd(a,b)

We have a similar base case:

With \gcd(a,0)=a\gcd(a, 0) = agcd(a,0)=a, it's obvious that:

1⋅a+0⋅0=a1 \cdot a + 0 \cdot 0 = a1⋅a+0⋅0=a

Now, for the general case, we decompose aaa into qb+rqb + rqb+r, like before.

Let's say we've found x,yx, yx,y such that:

xb+yr=\gcd(b,r)=\gcd(a,b)xb + yr = \gcd(b, r) = \gcd(a, b)xb+yr=gcd(b,r)=gcd(a,b)

But, we can write rrr as:

r=a−qbr = a - qbr=a−qb

Which gives us:

\gcd(a,b)=xb+yr=xb+y(a−qb)=ya+(x−qy)b\gcd(a, b) = xb + yr = xb + y(a - qb) = ya + (x - qy)bgcd(a,b)=xb+yr=xb+y(a−qb)=ya+(x−qy)b

So (x,y)(x, y)(x,y) becomes (y,(x−q))(y, (x - q))(y,(x−q))

We can write out a similar algorithm pretty easily:

bezout :: Integer -> Integer -> (Integer, Integer)
bezout a b | a < b = let (x, y) = bezout b a in (y, x)
bezout a 0 = (1, 0)
bezout a b =
  let q = div a b
      r = mod a b
      (x, y) = bezout b r
  in (y, x - q * y)

The Chinese Remainder Theorem

We now have all the tools we need to tackle the CRT itself!

To set the stage again, our starting point is a system of kkk equations:

x≡a1modn1x≡a2modn2⋮x≡akmodnk\begin{aligned} x &\equiv a_1 \mod n_1 \cr x &\equiv a_2 \mod n_2 \cr \vdots \cr x &\equiv a_k \mod n_k \end{aligned}xx⋮x​≡a1​modn1​≡a2​modn2​≡ak​modnk​​

and our goal here, is to find some integer x∈Zx \in \mathbb{Z}x∈Z satisfying each of these equations.

The CRT applies in the case where n1,n2,…,nkn_1, n_2, \ldots, n_kn1​,n2​,…,nk​ are all coprime.

This means that:

\gcd(ni,nj)=1\gcd(n_i, n_j) = 1gcd(ni​,nj​)=1

for any pair i,ji, ji,j.

Two Equations

Let's first look at the case where we have just two equations:

x≡a1modn1x≡b2modn2\begin{aligned} x &\equiv a_1 \mod n_1 \cr x &\equiv b_2 \mod n_2 \end{aligned}xx​≡a1​modn1​≡b2​modn2​​

At first this might seem tricky, but remember that we assumed that n1n_1n1​ and n2n_2n2​ are coprime. This means that \gcd(n1,n2)=1\gcd(n_1, n_2) = 1gcd(n1​,n2​)=1. Let's try using Bézout's algorithm!

This gives us m1,m2m_1, m_2m1​,m2​ such that:

m1n1+m2n2=1m_1 n_1 + m_2 n_2 = 1m1​n1​+m2​n2​=1

Notice if that if we look at this equation modulo n1n_1n1​, we get:

m2n2≡1modn1m_2 n_2 \equiv 1 \mod n_1m2​n2​≡1modn1​

This is because modulo n1n_1n1​, the extra multiple m1n1m_1 n_1m1​n1​ simply disappears

But multiplying anything by 111 gives us back that same number, even modulo n1n_1n1​.

This means that:

a1m2n2≡a1modn1a_1 m_2 n_2 \equiv a_1 \mod n_1a1​m2​n2​≡a1​modn1​

But hey, that's one of the things we wanted to be true!

We can apply a similar argument to show that:

a2m1n1≡a2modn2a_2 m_1 n_1 \equiv a_2 \mod n_2a2​m1​n1​≡a2​modn2​

So, we have a solution for each part, but how can we combine them to get a solution for both parts at once?

We just add them together!

a1m2n2+a2m1n1a_1 m_2 n_2 + a_2 m_1 n_1a1​m2​n2​+a2​m1​n1​

This works, because modulo n1n_1n1​, the right part disappears, and we get a1m2n2a_1 m_2 n_2a1​m2​n2​, which is a1a_1a1​, as we saw before.

Similarly, modulo n2n_2n2​, the left part disappears, and we get a2m1n1a_2 m_1 n_1a2​m1​n1​, which also works out to a2a_2a2​, as we saw before.

This gives us a solution for the case of two equations.

Generalizing

Let's say we have kkk equations, instead of just 222:

x≡a1modn1x≡a2modn2⋮x≡aNmodnN\begin{aligned} x &\equiv a_1 \mod n_1 \cr x &\equiv a_2 \mod n_2 \cr \vdots \cr x &\equiv a_N \mod n_N \end{aligned}xx⋮x​≡a1​modn1​≡a2​modn2​≡aN​modnN​​

What we're going to be doing is solving the first two equations, and then using that to produce a new system of N−1N - 1N−1 equations.

Let a1,2a_{1,2}a1,2​ be a solution to the first 222 equations, i.e.

a1,2≡a1modn1a1,2≡a2modn2\begin{aligned} a_{1,2} &\equiv a_1 \mod n_1 \cr a_{1,2} &\equiv a_2 \mod n_2 \cr \end{aligned}a1,2​a1,2​​≡a1​modn1​≡a2​modn2​​

What we can prove is:

x≡a1,2modn1n2x \equiv a_{1,2} \mod n_1 n_2x≡a1,2​modn1​n2​

if and only if

x≡a1modn1x≡a2modn2\begin{aligned} x &\equiv a_1 \mod n_1 \cr x &\equiv a_2 \mod n_2 \cr \end{aligned}xx​≡a1​modn1​≡a2​modn2​​

The first direction is simple. If x≡a1,2modn1n2x \equiv a_{1, 2} \mod n_1 n_2x≡a1,2​modn1​n2​, then x−a1,2x - a_{1, 2}x−a1,2​ is a multiple of n1n2n_1 n_2n1​n2​, i.e. it is kn1n2k n_1 n_2kn1​n2​ for some kkk. It's then clear that x−a1,2x - a_{1, 2}x−a1,2​ is also a multiple of n1n_1n1​, as well as n2n_2n2​.

This means that:

x≡a1,2modn1x≡a1,2modn2\begin{aligned} x &\equiv a_{1, 2} \mod n_1 \cr x &\equiv a_{1, 2} \mod n_2 \cr \end{aligned}xx​≡a1,2​modn1​≡a1,2​modn2​​

But, remember that a1,2a_{1, 2}a1,2​ is a solution to the first two equations, giving us:

x≡a1modn1x≡a2modn2\begin{aligned} x &\equiv a_1 \mod n_1 \cr x &\equiv a_2 \mod n_2 \cr \end{aligned}xx​≡a1​modn1​≡a2​modn2​​

□\square□

For the other direction, we assume that:

x≡a1modn1x≡a2modn2\begin{aligned} x &\equiv a_1 \mod n_1 \cr x &\equiv a_2 \mod n_2 \cr \end{aligned}xx​≡a1​modn1​≡a2​modn2​​

Using the fact that a1,2a_{1, 2}a1,2​ is a solution, once more we get:

x≡a1,2modn1x≡a1,2modn2\begin{aligned} x &\equiv a_{1, 2} \mod n_1 \cr x &\equiv a_{1, 2} \mod n_2 \cr \end{aligned}xx​≡a1,2​modn1​≡a1,2​modn2​​

This means that (x−a1,2)(x - a_{1, 2})(x−a1,2​) is a multiple of both n1n_1n1​ and n2n_2n2​,i.e.

x−a1,2=k1n1=k2n2x - a_{1, 2} = k_1 n_1 = k_2 n_2x−a1,2​=k1​n1​=k2​n2​

But, since \gcd(n1,n2)=1\gcd(n_1, n_2) = 1gcd(n1​,n2​)=1 we can show that k1=kn2k_1 = k n_2k1​=kn2​:

We can write k1n1k_1 n_1k1​n1​ as

k1(1−m2n2)=k2n2k_1 (1 - m_2 n_2) = k_2 n_2k1​(1−m2​n2​)=k2​n2​

using Bézout, but then we get:

k1=k1m2n2+k2n2=(k1m2n2+k2)n2k_1 = k_1 m_2 n_2 + k_2 n_2 = (k_1 m_2 n_2 + k_2) n_2k1​=k1​m2​n2​+k2​n2​=(k1​m2​n2​+k2​)n2​

showing that k1k_1k1​ is indeed a multiple of n2n_2n2​.

Thus we have:

x−a1,2=kn2n1x - a_{1, 2} = k n_2 n_1x−a1,2​=kn2​n1​

but this means that:

x≡a1,2modn1n2x \equiv a_{1, 2} \mod n_1 n_2x≡a1,2​modn1​n2​

as desired □\square□.

Since x≡a1,2modn1n2x \equiv a_{1, 2} \mod n_1 n_2x≡a1,2​modn1​n2​ is equivalent to solving both of the first two equations, we can simply replace both of those equations, to get:

x≡a1,2modn1n2x≡a3modn3⋮x≡aNmodnN\begin{aligned} x &\equiv a_{1, 2} \mod n_1 n_2 \cr x &\equiv a_3 \mod n_3 \cr \vdots \cr x &\equiv a_N \mod n_N \end{aligned}xx⋮x​≡a1,2​modn1​n2​≡a3​modn3​≡aN​modnN​​

We now have one less equation, so this works as a recursive rule.

As a base case, if we end up with a single equation:

x≡amodnx \equiv a \mod nx≡amodn

we simply choose aaa as our solution.

Concretely

Alrighty, that was a bit of a mouthful, but hopefully it'll be a bit more understandable if we write this out in code.

We have:

solve :: [(Integer, Integer)] -> Integer

as our signature. This function will return the smallest positive integer solving a list of (a,n)(a, n)(a,n) congruence pairs.

For zero equations, anything is a solution. So we just return 000

solve [] = 0

For a single equation, the solution is obvious:

solve [(a, n)] = fullMod a n

Note that we use the fullMod function we defined earlier, so that we get the smallest positive solution. a itself would be a solution, but we normalize it here to get the "nicest" version of it.

Now for two or more equations:

solve ((a1, n1) : (a2, n2) : rest) =
  let (m1, m2) = bezout n1 n2
      a12 = a1 * m2 * n2 + a2 * m1 * n1
  in solve ((a12, n1 * n2) : rest)

So, we solve the first two equations, using Bézout, to get:

a1,2=a1m2n2+a2m1n1a_{1,2} = a_1 m_2 n_2 + a_2 m_1 n_1a1,2​=a1​m2​n2​+a2​m1​n1​

And then we produce an equivalent equation x≡a1,2modn1n2x \equiv a_{1, 2} \mod n_1 n_2x≡a1,2​modn1​n2​, and continue solving, having replaced those two equations with this single one.
